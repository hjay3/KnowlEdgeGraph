The Hook System

pytest uses pluggy under the hood — a minimalist hook specification & calling library. Hooks are declared in pytest.hookspec; any plugin implementing a matching pytest_* function participates automatically.

conftest.py — hook anatomy python 

# Every conftest.py is implicitly a plugin.
# Hook implementations are just functions with matching names.

def pytest_configure(config):
    """Called after command-line options have been parsed.
    Register custom marks here to avoid PytestUnknownMarkWarning."""
    config.addinivalue_line("markers", "slow: mark test as slow to run")
    config.addinivalue_line("markers", "integration: requires live services")
    config.addinivalue_line("markers", "smoke: quick sanity gate")

def pytest_collection_modifyitems(config, items):
    """Mutate or reorder the collected test list.
    Classic use: auto-mark tests in certain paths."""
    for item in items:
        if "integration" in item.nodeid:
            item.add_marker(pytest.mark.integration)
        # Ensure slow tests run last — minimises CI tail latency
        if item.get_closest_marker("slow"):
            item.add_marker(pytest.mark.last)   # requires pytest-ordering

def pytest_runtest_makereport(item, call):
    """Intercept per-phase outcomes (setup/call/teardown).
    Attach phase outcome to item for use in fixtures."""
    if call.when == "call":
        item._call_outcome = call   # stash for fixture teardown use

Test Discovery

pytest traverses the filesystem starting from testpaths (or current dir) and collects anything matching the configured naming conventions. Understanding discovery prevents the two most common new-user pitfalls: import errors and duplicated test IDs.

Default discovery rules

Recurse into test_*.py / *_test.py files. Collect Test* classes (no __init__) and test_* functions/methods. Also collects unittest.TestCase subclasses.

--import-mode

prepend (legacy): inserts parent dirs into sys.path. importlib (recommended): unique import per file, no __init__.py needed, no path pollution. Set in pyproject.toml.

recommended project layout text 

myproject/
├── src/
│   └── myapp/
│       ├── __init__.py
│       ├── core.py
│       └── utils.py
├── tests/
│   ├── conftest.py          ← session-wide fixtures & hooks
│   ├── unit/
│   │   ├── conftest.py      ← unit-scope fixtures
│   │   ├── test_core.py
│   │   └── test_utils.py
│   ├── integration/
│   │   ├── conftest.py      ← db/service fixtures
│   │   └── test_api.py
│   └── e2e/
│       └── test_flows.py
├── pyproject.toml
└── pytest.ini               ← or [tool.pytest.ini_options] in above

Running Subsets — nodeids & expressions

$ pytest tests/unit/ # directory
$ pytest tests/unit/test_core.py # file
$ pytest tests/unit/test_core.py::test_add# specific function
$ pytest tests/unit/test_core.py::TestMath::test_mul# class method
$ pytest -k "add or subtract and not slow" # keyword expr
$ pytest -m "integration and not slow" # mark expr
$ pytest --collect-only -q # dry-run: show what would run
$ pytest --co -q -m smoke # collect smoke tests only

Assertion Introspection

pytest rewrites assert statements at import time using AST transformation. The rewritten bytecode captures sub-expression values, giving surgical failure messages without any helper infrastructure.

test_assertions.py — what good failures look like python 

import pytest

# ── Basic scalar ──────────────────────────────────────────────────
def test_scalar_failure():
    x = 42
    assert x == 43
    # FAILURE OUTPUT:
    # AssertionError: assert 42 == 43
    #  +  where 42 = x

# ── Collection membership ─────────────────────────────────────────
def test_membership():
    result = {"a": 1, "b": 2}
    assert "c" in result
    # AssertionError: assert 'c' in {'a': 1, 'b': 2}

# ── Complex expressions ───────────────────────────────────────────
def test_complex():
    data = [1, 2, 3, 4]
    assert len(data) == 3
    # AssertionError: assert 4 == 3
    #  +  where 4 = len([1, 2, 3, 4])

# ── Custom failure messages — use sparingly ───────────────────────
def test_with_message():
    user_id = 99
    assert user_id > 0, f"Expected positive user_id, got {user_id!r}"

# ── Exception testing (best practice) ────────────────────────────
def test_raises():
    with pytest.raises(ValueError, match=r"invalid literal.*'abc'"):
        int("abc")

def test_raises_captures():
    with pytest.raises(KeyError) as exc_info:
        d = {}; d["missing"]
    assert exc_info.value.args[0] == "missing"

# ── Approximate equality (floats) ─────────────────────────────────
def test_approx():
    assert 0.1 + 0.2 == pytest.approx(0.3)
    assert 0.1 + 0.2 == pytest.approx(0.3, rel=1e-6)   # relative tol
    assert [0.1, 0.2] == pytest.approx([0.1, 0.2], abs=1e-9)

# ── Warning assertions ────────────────────────────────────────────
def test_warning():
    import warnings
    with pytest.warns(DeprecationWarning, match="use new_api"):
        warnings.warn("use new_api", DeprecationWarning)

# ── Assertion rewriting hooks — extend for custom objects ─────────
# In conftest.py:
# pytest.register_assert_rewrite("myapp.testing")
# Any module registered here gets assertion rewriting too.
Pythonic assertion style

Never write assertEqual, assertTrue, assertIsNone — these are unittest relics. A plain assert with pytest's rewriter is more informative. The only exception: pytest.approx for floats and pytest.raises / pytest.warns for exception/warning capture.

Core Fixture System

Fixtures are pytest's superpower: composable, lazily-resolved dependency injections with deterministic teardown. They replace setUp/tearDown with something far more expressive.

tests/conftest.py — foundational patterns python 

import pytest
import tempfile, os, contextlib
from pathlib import Path
from unittest.mock import MagicMock, patch

# ── Basic fixture (function scope) ────────────────────────────────
@pytest.fixture
def sample_user():
    """Minimal user dict. Teardown not needed for pure data."""
    return {"id": 1, "name": "Ada Lovelace", "role": "admin"}

# ── Yield fixture: setup / teardown ──────────────────────────────
@pytest.fixture
def temp_dir():
    """Temporary directory that is always cleaned up."""
    with tempfile.TemporaryDirectory() as d:
        yield Path(d)           # ← test body runs here
        # TemporaryDirectory.__exit__ runs teardown automatically

# ── Explicit teardown via finaliser ──────────────────────────────
@pytest.fixture
def database_connection(db_url):
    conn = create_connection(db_url)       # hypothetical
    conn.begin()
    try:
        yield conn
    finally:
        conn.rollback()     # always rollback — tests are isolated
        conn.close()

# ── Fixture consuming other fixtures ─────────────────────────────
@pytest.fixture
def authenticated_client(client, sample_user):
    """Compose fixtures — pytest resolves the graph automatically."""
    client.login(username=sample_user["name"], password="secret")
    yield client
    client.logout()

# ── Autouse: applied to every test in scope ───────────────────────
@pytest.fixture(autouse=True)
def reset_environ():
    """Guarantee clean env-var state around every test."""
    original = os.environ.copy()
    yield
    os.environ.clear()
    os.environ.update(original)

# ── Fixture with params (generates multiple test instances) ───────
@pytest.fixture(params=["sqlite", "postgres", "mysql"])
def db_engine(request):
    engine = request.param
    # request.param gives the current parameter value
    yield engine

# ── Parametrized fixture with IDs ─────────────────────────────────
SETTINGS = [
    pytest.param({"debug": True,  "cache": False}, id="debug-nocache"),
    pytest.param({"debug": False, "cache": True},  id="prod-cached"),
    pytest.param({"debug": False, "cache": False}, id="prod-nocache"),
]

@pytest.fixture(params=SETTINGS)
def app_settings(request):
    return request.param

# ── request object — the fixture's introspection API ─────────────
@pytest.fixture
def tmp_file(request, tmp_path):
    """Use request.param when called with indirect parametrization."""
    filename = getattr(request, "param", "default.txt")
    p = tmp_path / filename
    p.write_text("")
    return p

# ── Accessing test metadata inside a fixture ─────────────────────
@pytest.fixture
def log_test_info(request):
    """Know which test is running, its markers, and module."""
    test_name = request.node.name
    module    = request.node.module.__name__
    markers   = [m.name for m in request.node.iter_markers()]
    print(f"\n[fixture] running {module}::{test_name} marks={markers}")
Teardown guarantee

A yield fixture's teardown block runs even if the test raises — pytest ensures it. But if an earlier fixture in the dependency graph fails during setup, later fixtures won't be set up (or torn down). Design fixture graphs to be resilient to partial setup by wrapping teardown in try/finally.

tests/conftest.py — scope ladder python 

import pytest
import docker                      # pip install docker
import sqlalchemy as sa

# ── Session scope: start docker postgres once per CI run ──────────
@pytest.fixture(scope="session")
def postgres_container():
    """Start a real Postgres container for the whole session."""
    client = docker.from_env()
    container = client.containers.run(
        "postgres:16-alpine",
        environment={"POSTGRES_PASSWORD": "secret", "POSTGRES_DB": "testdb"},
        ports={"5432/tcp": None},   # random host port
        detach=True,
        remove=True,
    )
    container.reload()
    host_port = container.ports["5432/tcp"][0]["HostPort"]
    # Wait for pg to be ready
    import time
    for _ in range(30):
        try:
            eng = sa.create_engine(f"postgresql://postgres:secret@localhost:{host_port}/testdb")
            with eng.connect(): break
        except Exception:
            time.sleep(1)
    yield f"postgresql://postgres:secret@localhost:{host_port}/testdb"
    container.stop()

# ── Session scope: create schema once ────────────────────────────
@pytest.fixture(scope="session")
def db_engine(postgres_container):
    engine = sa.create_engine(postgres_container, pool_pre_ping=True)
    Base.metadata.create_all(engine)     # hypothetical ORM base
    yield engine
    Base.metadata.drop_all(engine)
    engine.dispose()

# ── Module scope: one connection pool per test file ───────────────
@pytest.fixture(scope="module")
def db_session_factory(db_engine):
    factory = sessionmaker(bind=db_engine)
    yield factory

# ── Function scope: isolated transaction per test (fast!) ─────────
@pytest.fixture
def db(db_session_factory):
    """Each test runs in a transaction that is always rolled back.
    Reads committed data from session fixtures but never pollutes."""
    session = db_session_factory()
    session.begin_nested()             # savepoint
    try:
        yield session
    finally:
        session.rollback()
        session.close()

# ── Dynamic scope: determined at runtime ──────────────────────────
def determine_scope(fixture_name, config):
    """Use --integration flag to upgrade fixture to session scope."""
    if config.getoption("--integration", default=False):
        return "session"
    return "function"

@pytest.fixture(scope=determine_scope)
def external_api_client(request):
    """Mock locally, real in integration runs."""
    if request.scope == "function":
        yield MagicMock()        # fast unit default
    else:
        client = RealAPIClient(base_url=os.environ["API_URL"])
        yield client
        client.close()
Scope mismatch: the silent killer

A function-scoped fixture cannot depend on a session-scoped one without pytest raising ScopeMismatch. The rule: a fixture can only consume fixtures of equal or broader scope. Violating this is caught at collection time — but confusingly only when the offending combination is actually requested.

conftest.py files are auto-loaded plugins scoped to their directory tree. pytest walks the filesystem upward from each collected test, loading every conftest.py it encounters — nearest wins.

import pytest
import logging

# ── Custom CLI options ────────────────────────────────────────────
def pytest_addoption(parser):
    """Add project-specific CLI flags."""
    parser.addoption(
        "--integration", action="store_true", default=False,
        help="run integration tests against real services"
    )
    parser.addoption(
        "--env", choices=["dev", "staging", "prod"], default="dev",
        help="target environment for integration tests"
    )

@pytest.fixture(scope="session")
def env(request):
    return request.config.getoption("--env")

@pytest.fixture(scope="session")
def run_integration(request):
    return request.config.getoption("--integration")

# ── Skip integration unless flag given ───────────────────────────
def pytest_collection_modifyitems(config, items):
    if not config.getoption("--integration", default=False):
        skip = pytest.mark.skip(reason="pass --integration to run")
        for item in items:
            if item.get_closest_marker("integration"):
                item.add_marker(skip)

# ── Capture logs per-test automatically ──────────────────────────
@pytest.fixture(autouse=True)
def capture_logs(caplog):

Factory Fixtures

factory pattern — flexible test data python 

import pytest
from dataclasses import dataclass, field
from typing import Any

@dataclass
class User:
    id: int
    name: str
    email: str
    role: str = "viewer"
    metadata: dict = field(default_factory=dict)

# ── Factory fixture: returns a callable ───────────────────────────
@pytest.fixture
def make_user():
    """Factory: each call creates an independent User.
    Tests can customise fields without fixture explosion."""
    created = []

    def _factory(name="Alice", email=None, role="viewer", **kwargs) -> User:
        user = User(
            id=len(created) + 1,
            name=name,
            email=email or f"{name.lower()}@example.com",
            role=role,
            **kwargs,
        )
        created.append(user)
        return user

    yield _factory
    # teardown: could persist cleanup of created objects

# ── Usage in tests ────────────────────────────────────────────────
def test_admin_can_delete(make_user):
    admin = make_user(role="admin")
    viewer = make_user(name="Bob")
    assert admin.role == "admin"
    assert viewer.id == 2           # independent IDs

def test_user_email_defaults(make_user):
    user = make_user(name="Charlie")
    assert user.email == "charlie@example.com"

import pytest
import math

# ── Single argument ───────────────────────────────────────────────
@pytest.mark.parametrize("n,expected", [
    (0,  1),
    (1,  1),
    (5,  120),
    (10, 3628800),
])
def test_factorial(n, expected):
    assert math.factorial(n) == expected

# ── Named params with pytest.param ───────────────────────────────
@pytest.mark.parametrize("payload,status_code", [
    pytest.param({"name": ""},      400, id="empty-name"),
    pytest.param({"name": "x"*300}, 422, id="name-too-long"),
    pytest.param({"name": "Valid"}, 201, id="valid"),
    pytest.param(None,              400, id="null-body"),
])
def test_create_user_validation(client, payload, status_code):
    resp = client.post("/users", json=payload)
    assert resp.status_code == status_code

# ── Stacked parametrize: Cartesian product ────────────────────────
# Generates 2 × 3 = 6 test cases automatically
@pytest.mark.parametrize("protocol", ["http", "https"])
@pytest.mark.parametrize("method",   ["GET", "POST", "PUT"])
def test_request_routing(protocol, method):
    # 6 combinations: http+GET, http+POST, http+PUT, https+GET, ...
    url = f"{protocol}://api.example.com/resource"
    assert url.startswith(protocol)

# ── Conditional skip/xfail inside params ─────────────────────────
import sys

@pytest.mark.parametrize("algo", [
    "sha256",
    pytest.param("sha3_256", marks=pytest.mark.skipif(

    )),
    pytest.param("blake2b", marks=pytest.mark.xfail(

    )),
])
def test_hash_algo(algo):

    h = hashlib.new(algo, b"data").hexdigest()

from datetime import date

@pytest.mark.parametrize("start,end,expected_days", [

The Mark System

test_marks.py — the full mark vocabulary python 

import pytest, sys, os

# ── skip: unconditional ───────────────────────────────────────────
@pytest.mark.skip(reason="WIP: not implemented yet")
def test_future_feature():
    ...

# ── skipif: conditional ───────────────────────────────────────────
@pytest.mark.skipif(sys.platform == "win32", reason="posix-only API")
def test_unix_permissions():
    assert oct(os.stat("/tmp").st_mode)[-3:] in {"755", "1777"}

# ── xfail: expected failure ───────────────────────────────────────
@pytest.mark.xfail(reason="upstream bug GH-1234", strict=False)
def test_buggy_upstream():
    # If this PASSES (XPASS), it's reported but not an error
    # strict=True: XPASS becomes an error — great for bug tracking
    from upstream import broken_function
    assert broken_function() == 42

@pytest.mark.xfail(strict=True, reason="must remain broken until patch")
def test_known_regression():
    # strict=True means: if this starts passing, FAIL the suite.
    # Perfect for preventing accidental regressions from passing silently.
    raise NotImplementedError

# ── xfail with condition and raises ──────────────────────────────
@pytest.mark.xfail(
    sys.version_info < (3, 11),
    reason="ExceptionGroup requires 3.11+",
    raises=AttributeError,
)
def test_exception_group():
    raise ExceptionGroup("multi", [ValueError("a"), TypeError("b")])

# ── usefixtures: apply fixtures without receiving them ────────────
@pytest.mark.usefixtures("reset_environ", "mock_http")
class TestAPIClient:
    def test_get(self, client):
        resp = client.get("/health")
        assert resp.status_code == 200

# ── Custom marks with metadata ────────────────────────────────────
# Declared in pyproject.toml to avoid warnings; used here:
@pytest.mark.slow

Live & Production Debugging

The hardest bugs are the ones that only reproduce in production. pytest can be wielded to systematically reproduce, narrow, and confirm production incidents — without deploying code changes.

Strategy: Incident-Driven Test Writing

The Golden Rule

Before fixing a production bug: write a failing test that reproduces it. Only then fix. Run the test. Green means fixed. The test becomes a permanent regression guard. This is not TDD religion — it's engineering discipline under fire.

@pytest.hookimpl(hookwrapper=True)
def pytest_runtest_protocol(item, nextitem):
    """Wrap the entire test protocol. hookwrapper=True gives before/after."""
    start = time.perf_counter()
    yield                           # pytest runs setup/call/teardown here
    duration = time.perf_counter() - start
    if item.config._myplugin_verbose:
        print(f"\n  [{duration:.3f}s] {item.nodeid}")

    """Intercept the per-phase report (setup/call/teardown)."""
    outcome = yield
    report  = outcome.get_result()
    # Attach phase reports to the item for fixture access:
    item.stash.setdefault("phases", {})[call.when] = report

Anti-patterns & Pitfalls

Over-mocking

Mocking everything produces tests that pass even when the code is wrong. Mock at the boundary (HTTP, DB, filesystem) — not internal functions. If you're mocking 10+ things, the test is testing the mocks, not the code.

Fixture soup

A fixture that does 12 things and has 8 dependencies is a smell. Split into composable single-responsibility fixtures. Each fixture should have exactly one reason to exist.

Shared mutable state

Session-scoped mutable objects shared across tests cause order-dependent failures. Either use immutable session fixtures or transaction rollbacks for function-scoped isolation.

assert in fixtures

Assertions inside fixtures produce confusing errors (test shows as ERROR, not FAIL). Assertions belong in tests. Fixtures should only set up state and yield it.

Undeclared marks

Without --strict-markers, a typo in a mark (@pytest.mark.integratoin) silently creates a new mark. Always use --strict-markers and declare all marks in pyproject.toml.

sleep() in tests

Never use time.sleep() in tests. It's brittle on slow CI. Use pytest-freezegun to mock time, or proper async event loops. If testing retry logic, mock the delay function.

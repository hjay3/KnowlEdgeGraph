Section 2 — Core Mechanics

The toolkit every 3D programmer uses every single day. Know these cold.

BASIC → EARLY INTERMEDIATE

The Compounding Power of Core Mechanics

The dot product, the cross product, and the 4×4 matrix are not separate tools — they are three faces of the same gem. The dot product measures alignment, the cross product manufactures perpendicularity, and the matrix composes both into an arbitrary transform. Master all three as a unified system, not as isolated formulas.

The Dot Product — Similarity in One Number

A·B = |A||B|cos(θ). When both vectors are unit length, the dot product IS the cosine of the angle between them. This single insight powers diffuse lighting, backface culling, specular highlights, and frustum culling.

Key insight: dot product is positive when vectors agree, zero when perpendicular, negative when opposing. The dashed line shows the shadow (projection) of A onto B.

// Diffuse lighting — one line! float diff = max(dot(normal, lightDir), 0.0); color *= diff; // Backface culling check if (dot(faceNormal, viewDir) < 0.0) discard();

The Cross Product — Birth of a New Direction

A×B produces a vector perpendicular to both A and B. Its magnitude equals |A||B|sin(θ) — the area of the parallelogram formed by A and B. Fundamental for computing normals, tangent spaces, and orientation.

Direction follows the right-hand rule. Order matters: A×B = −(B×A). Reversing the order flips the normal — which is exactly how winding order determines front/back faces.

// Normal from triangle verts vec3 edge1 = B - A; vec3 edge2 = C - A; vec3 normal = normalize(cross(edge1, edge2)); // Area of triangle = 0.5 * |cross| float area = 0.5 * length(cross(edge1, edge2));

Matrix Transforms — Linear Algebra Made Visual

A 3×3 matrix transforms a vector. Its columns tell you exactly where the basis vectors (î, ĵ, k̂) land after the transform. Composition of transforms = matrix multiplication. The entire 3D pipeline is a chain of matrix multiplications.

The animated grid shows your space being stretched and rotated simultaneously — this is what a matrix does to every point in space at once.

// MVP transform (order matters!) gl_Position = proj * view * model * vec4(pos, 1.0); // Columns = where basis vectors land // mat[0] = new î, mat[1] = new ĵ

Perspective Projection — Depth Creates Illusion

Dividing X and Y by Z gives perspective foreshortening — objects farther away appear smaller. The projection matrix encodes the field of view, aspect ratio, near/far planes and maps everything to NDC space [-1,1]³.

Why does dividing by Z work? Because the further something is, the smaller the angle it subtends. Division by depth is the mathematical encoding of that angular shrinkage.

x_ndc = x / (z * tan(fov/2)); y_ndc = y / (z * tan(fov/2)); // z is preserved (non-linearly) for depth test
